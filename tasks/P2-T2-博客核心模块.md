## 详细步骤

### 1. 博客 CRUD 与标签关联

```java
// 新建/编辑博客时维护中间表
@Transactional
public void saveBlog(BlogSaveRequest req) {
    TBlog blog = BlogConverter.INSTANCE.toEntity(req);

    if (req.getId() == null) {
        blogMapper.insert(blog);
    } else {
        blogMapper.updateById(blog);
        // 清除旧标签关联
        blogTagMapper.delete(new LambdaQueryWrapper<TBlogTag>()
            .eq(TBlogTag::getBlogId, req.getId()));
    }

    // 插入新标签关联
    if (!CollectionUtils.isEmpty(req.getTagIds())) {
        List<TBlogTag> relations = req.getTagIds().stream()
            .map(tagId -> new TBlogTag(blog.getId(), tagId))
            .collect(Collectors.toList());
        blogTagMapper.batchInsert(relations);
    }

    // 如果是发布操作，清除详情缓存
    if (req.getStatus() == 1) {
        redisTemplate.delete(RedisKeys.BLOG_DETAIL + blog.getId());
    }
}
```

### 2. 博客详情 + 浏览量

```java
public BlogDetailVO getBlogDetail(Long id) {
    // 先查缓存
    Object cached = redisTemplate.opsForValue().get(RedisKeys.BLOG_DETAIL + id);
    if (cached != null) {
        // 异步增加浏览量（不阻塞响应）
        incrementViewAsync(id);
        return (BlogDetailVO) cached;
    }

    TBlog blog = blogMapper.selectById(id);
    if (blog == null || blog.getStatus() != 1) {
        throw new BusinessException(ErrorCode.BLOG_NOT_FOUND);
    }

    BlogDetailVO vo = buildDetailVO(blog);

    // 写入缓存，TTL 10 分钟
    redisTemplate.opsForValue().set(
        RedisKeys.BLOG_DETAIL + id, vo, 10, TimeUnit.MINUTES
    );

    incrementViewAsync(id);
    return vo;
}

// 异步累加 Redis 浏览量
@Async
public void incrementViewAsync(Long blogId) {
    redisTemplate.opsForValue().increment(RedisKeys.BLOG_VIEWS + blogId);
}
```

### 3. 浏览量定时同步至 MySQL

```java
@Component
@RequiredArgsConstructor
public class ViewSyncTask {

    private final RedisTemplate<String, Object> redisTemplate;
    private final BlogMapper blogMapper;

    // 每 5 分钟同步一次
    @Scheduled(fixedDelay = 5 * 60 * 1000)
    public void syncViewsToDb() {
        Set<String> keys = redisTemplate.keys(RedisKeys.BLOG_VIEWS + "*");
        if (CollectionUtils.isEmpty(keys)) return;

        for (String key : keys) {
            Long blogId = Long.parseLong(key.replace(RedisKeys.BLOG_VIEWS, ""));
            Object value = redisTemplate.opsForValue().getAndDelete(key);
            if (value != null) {
                long increment = Long.parseLong(value.toString());
                blogMapper.incrementViews(blogId, increment);
            }
        }
    }
}
```

```xml
<!-- BlogMapper.xml -->
<update id="incrementViews">
    UPDATE t_blog SET views = views + #{increment} WHERE id = #{blogId}
</update>
```

### 4. 博客点赞（IP 去重）

```java
public void likeBlog(Long blogId, String clientIp) {
    String key = RedisKeys.BLOG_LIKE + blogId + ":" + clientIp;
    Boolean exists = redisTemplate.hasKey(key);
    if (Boolean.TRUE.equals(exists)) {
        throw new BusinessException(400, "今天已经点过赞了");
    }
    // 记录点赞标记，24h 过期
    redisTemplate.opsForValue().set(key, "1", 24, TimeUnit.HOURS);
    // 直接更新数据库（点赞不需要高频，直接落库）
    blogMapper.incrementLikes(blogId);
}
```

### 5. 多条件筛选分页（动态 SQL）

```xml
<!-- BlogMapper.xml -->
<select id="selectBlogPage" resultType="BlogListVO">
    SELECT b.id, b.title, b.cover_img, b.summary, b.views, b.likes,
           b.status, b.is_top, b.created_at,
           c.name AS category_name,
           u.nickname AS author_name
    FROM t_blog b
    LEFT JOIN t_category c ON b.category_id = c.id
    LEFT JOIN t_user u ON b.user_id = u.id
    <where>
        <if test="status != null">AND b.status = #{status}</if>
        <if test="categoryId != null">AND b.category_id = #{categoryId}</if>
        <if test="tagId != null">
            AND EXISTS (
                SELECT 1 FROM t_blog_tag bt WHERE bt.blog_id = b.id AND bt.tag_id = #{tagId}
            )
        </if>
        <if test="keyword != null and keyword != ''">
            AND (b.title LIKE CONCAT('%', #{keyword}, '%')
              OR b.summary LIKE CONCAT('%', #{keyword}, '%'))
        </if>
    </where>
    ORDER BY b.is_top DESC, b.created_at DESC
</select>
```

### 6. 归档时间轴

```xml
<select id="selectArchive" resultType="ArchiveVO">
    SELECT
        YEAR(created_at) AS year,
        MONTH(created_at) AS month,
        COUNT(id) AS count,
        JSON_ARRAYAGG(
            JSON_OBJECT('id', id, 'title', title, 'createdAt', created_at)
        ) AS blogs
    FROM t_blog
    WHERE status = 1
    GROUP BY YEAR(created_at), MONTH(created_at)
    ORDER BY year DESC, month DESC
</select>
```

## 验收标准

- [ ] `GET /api/v1/blogs?page=1&size=10` 返回分页博客列表，字段包含 `categoryName`、`tags`
- [ ] `GET /api/v1/blogs/{id}` 返回博客详情，同时浏览量 +1（可刷新两次验证 views 字段变化）
- [ ] 博客详情接口第二次请求响应时间 < 第一次（命中 Redis 缓存）
- [ ] `POST /api/v1/blogs/{id}/like` 同一 IP 当天第二次点赞返回错误
- [ ] 后台可正常新建博客并关联多个标签，`t_blog_tag` 中间表有对应记录
- [ ] 编辑博客重新保存后旧标签正确替换（中间表无冗余数据）
- [ ] 博客发布后 Redis 详情缓存被清除（下次请求重新建立缓存）
- [ ] `GET /api/v1/blogs/archive` 返回按年月分组的树形结构
- [ ] 等待 5 分钟后 MySQL `t_blog.views` 与 Redis 累计值同步（或手动触发定时任务验证）
- [ ] 开启 `@EnableScheduling` 注解，定时任务在启动日志中可见

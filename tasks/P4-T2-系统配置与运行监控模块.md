## 任务目标

新增「系统配置」与「运行监控」能力，支持后台在线修改关键配置，并可视化查看服务器/服务状态、缓存状态，提供基础运维操作能力，降低手工改配置与排障成本。

---

## 模块一：系统配置中心（2d）

### 功能范围

- [ ] 系统参数在线维护（键值配置）
- [ ] 分组管理：`site`、`seo`、`comment`、`upload`、`feature`
- [ ] 参数类型支持：`string`、`number`、`boolean`、`json`
- [ ] 支持参数描述、默认值、是否可编辑标记
- [ ] 修改后写入 Redis 缓存并广播刷新（或本地缓存失效）

### 后端建议设计

**数据表：`t_system_config`**

- `id`、`config_group`、`config_key`、`config_value`
- `value_type`、`description`、`editable`
- `created_at`、`updated_at`
- 唯一索引：`uniq_group_key(config_group, config_key)`

**接口建议：**

- `GET /admin/v1/system/config/group/{group}`
- `GET /admin/v1/system/config/item?group=&key=`
- `PUT /admin/v1/system/config/item`
- `POST /admin/v1/system/config/reload`（可选）

### 关键规则

- [ ] 非 `editable=1` 的配置不可在后台修改
- [ ] `json` 类型保存前必须校验 JSON 合法性
- [ ] 敏感键（如密钥）仅支持脱敏展示，不回传明文

---

## 模块二：服务器与服务状态监控（2d）

### 功能范围

- [ ] 服务器资源：CPU、内存、磁盘使用率
- [ ] JVM 状态：堆内存、线程数、GC 次数/耗时
- [ ] 服务健康：`server`、`mysql`、`redis` 状态
- [ ] 启动时长、当前版本号、构建时间展示

### 实现建议

- [ ] 启用 Spring Boot Actuator：`health`、`info`、`metrics`
- [ ] 封装 `GET /admin/v1/system/monitor/overview`
- [ ] 返回统一结构，时间字段格式：`yyyy-MM-dd HH:mm:ss`
- [ ] 健康状态统一枚举：`UP` / `DEGRADED` / `DOWN`

### 前端展示建议

- [ ] 概览卡片（CPU/内存/磁盘）
- [ ] 趋势图（近 30 分钟采样，按 1 分钟粒度）
- [ ] 服务状态列表（带最后探测时间）

---

## 模块三：缓存监控与运维操作（1.5d）

### 功能范围

- [ ] Redis 基础信息：版本、连接数、已用内存、命中率
- [ ] 业务缓存统计：按前缀统计 key 数量（如 `blog:*`、`category:*`）
- [ ] 缓存操作：按前缀清理、全量清理（高危二次确认）

### 接口建议

- `GET /admin/v1/system/cache/stats`
- `POST /admin/v1/system/cache/evict-prefix`
- `POST /admin/v1/system/cache/clear-all`

### 安全与审计

- [ ] 仅 `ROLE_ADMIN` 可访问
- [ ] 清理类操作记录操作日志（操作人、时间、范围）
- [ ] `clear-all` 需二次确认并限制频率（防误触）

---

## 模块四：后台页面与交互（0.5d）

### 页面建议

- [ ] 菜单新增：`系统配置`、`运行监控`
- [ ] 系统配置页：分组 Tab + 表格编辑 + 表单校验
- [ ] 运行监控页：状态卡 + 图表 + 缓存操作区

### 交互要求

- [ ] 加载中骨架或 Loading
- [ ] 失败态错误提示 + 一键重试
- [ ] 高危操作弹窗明确影响范围

---

## 验收标准

### 系统配置

- [ ] 后台可编辑 `site.title`、`comment.autoAudit` 等配置并生效
- [ ] 非可编辑配置被拒绝修改，返回明确错误
- [ ] JSON 配置非法时阻止保存并提示原因

### 运行监控

- [ ] 可看到 CPU/内存/磁盘实时值
- [ ] MySQL、Redis 异常时状态正确显示为 `DOWN` 或 `DEGRADED`
- [ ] 页面刷新不白屏，接口异常有可读提示

### 缓存监控

- [ ] 可查看 Redis 命中率与 key 统计
- [ ] 按前缀清理后统计值变化正确
- [ ] 全量清理有二次确认并写入审计日志

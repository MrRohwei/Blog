## 开发规范速查

### 分支策略
```
main      → 生产（只接受 PR）
dev       → 开发主分支
feat/xxx  → 功能分支
fix/xxx   → 修复分支
```

### 提交格式
```
feat: 新功能
fix: 修复 BUG
refactor: 重构
docs: 文档
style: 格式
test: 测试
```

### 接口规范
- 响应：`{ "code": 200, "message": "success", "data": {} }`
- 分页参数：`?page=1&size=10`（从第 1 页开始）
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 字段命名：`camelCase`
